-o选项允许用户指示输出文件名的模板。

基本用法是在下载单个文件时不设置任何模板参数，例如youtube-dl -o funny_video.flv“ https：// some / video”。 但是，它可能包含特殊的序列，这些序列将在下载每个视频时被替换。 可以根据python字符串格式化操作对特殊序列进行格式化。 例如，％（NAME）s或％（NAME）05d。 为了明确起见，这是一个百分号，后跟一个括号，然后是格式化操作。 允许的名称以及序列类型为：

  • id(string)：视频标识符
  • title(string)：视频标题
  • url(string)：视频URL
  • ext(string)：视频文件扩展名
  • alt_title(string)：视频的辅助标题
  • display_id(string)：视频的备用标识符
  • uploader(string)：视频上传者的全名
  • license(string)：视频所许可的许可证名称
  • creator(string)：视频的创建者
  • release_date(string)：视频发布的日期（YYYYMMDD）
  • timestamp(numeric)：视频可用时的UNIX时间戳
  • upload_date(string)：视频上传日期（YYYYMMDD）
  • uploader_id(string)：视频上传者的昵称或ID
  • channel(string)：上传视频的频道的全名
  • channel_id(string)：通道的ID
  • location(string)：录制视频的实际位置
  • duration(numeric)：视频长度（以秒为单位）
  • view_count(numeric)：在平台上观看了多少用户
  • like_count(numeric)：视频的正面评价数
  • dislike_count(numeric)：视频的负面评价数
  • repost_count(numeric)：视频的转发次数
  • average_rating(numeric)：用户给出的平均评分，使用的比例取决于网页
  • comment_count(numeric)：视频评论数
  • age_limit(numeric)：视频的年龄限制（年）
  • is_live（布尔值）：该视频是实时流还是固定长度的视频
  • start_time(numeric)：复制开始的时间（以秒为单位），如URL中所指定
  • end_time(numeric)：复制应结束的时间（以秒为单位），如URL中所指定
  • format(string)：格式的易于理解的描述
  • format_id(string)：--format指定的格式代码
  • format_note(string)：有关格式的其他信息
  • width(numeric)：视频的宽度
  • height(numeric)：视频的高度
  • resolution(string)：宽度和高度的文字描述
  • tbr(numeric)：音频和视频的平均比特率，单位为KBit / s
  • abr(numeric)：平均音频比特率，单位为KBit / s
  • acodec(string)：使用中的音频编解码器的名称
  • asr(numeric)：音频采样率，以赫兹为单位
  • vbr(numeric)：平均视频比特率，单位为KBit / s
  • fps(numeric)：帧频
  • vcodec(string)：正在使用的视频编解码器的名称
  • container(string)：容器格式的名称
  • filesize(numeric)：字节数（如果事先知道）
  • filesize_approx(numeric)：估计的字节数
  • protocol(string)：将用于实际下载的协议
  • extractor(string)：提取器的名称
  • extractor_key(string)：提取器的密钥名称
  • epoch(numeric)：创建文件时的Unix纪元
  • autonumber(numeric)：每次下载都会增加的编号，从--autonumber-start开始
  • playlist(string)：包含视频的播放列表的名称或ID
  • playlist_index(numeric)：播放列表中视频的索引，根据播放列表的总长度填充前导零
  • playlist_id(string)：播放列表标识符
  • playlist_title(string)：播放列表标题
  • playlist_uploader(string)：播放列表上传者的全名
  • playlist_uploader_id(string)：播放列表上传者的昵称或ID

适用于属于某些逻辑章节的视频：

  • chapter(string)：视频所属章节的名称或标题
  • chapter_number(numeric)：视频所属章节的编号
  • chapter_id(string)：视频所属章节的ID

适用于某些系列或节目的一集的视频：

  • series(string)：视频片段所属的系列或节目的标题
  • season(string)：视频片段所属的季节的标题
  • season_number(numeric)：视频片段所属的季节数
  • season_id(string)：视频片段所属的季节ID
  • episode (string)：视频剧集的标题
  • episode_number(numeric)：一个季节中的视频节目编号
  • episode_id(string)：视频剧集的ID

可用于作为曲目或音乐专辑一部分的媒体：

  • track(string)：轨道的标题
  • track_number(numeric)：专辑或光盘中轨道的编号
  • track_id(string)：轨道的ID
  • 艺术家(string)：曲目的艺术家
  • 流派(string)：曲目的流派
  • 专辑(string)：曲目所属专辑的标题
  • album_type(string)：相册的类型
  • album_artist(string)：出现在专辑中的所有艺术家的列表
  • disc_number(numeric)：轨道所属的光盘或其他物理介质的编号
  • release_year(numeric)：专辑发行的年份（YYYY）

格式选择

默认情况下，youtube-dl尝试下载最佳质量，即，如果您想要最佳质量，则不需要通过任何特殊选项，youtube-dl会默认为您猜测。 但是有时您可能想以其他格式下载，例如，在连接速度较慢或断断续续时。 实现此目的的关键机制是所谓的格式选择，您可以基于该机制显式指定所需的格式，基于某个或多个条件选择格式，设置优先级等。 格式选择的一般语法是--format FORMAT或更短的-f FORMAT，其中FORMAT是选择器表达式，即描述您要下载的一种或多种格式的表达式。

tl;dr: navigate me to examples.

最简单的情况是请求特定格式，例如，使用-f 22可以下载格式代码等于22的格式。您可以使用--list-formats或-F获取特定视频的可用格式代码列表。 请注意，这些格式代码是提取器特定的。 您还可以使用文件扩展名（当前支持3gp，aac，flv，m4a，mp3，mp4，ogg，wav，webm）下载作为单个文件的特定文件扩展名的最佳格式，例如 -f webm将以webm扩展名作为单个文件下载最佳质量格式。 您还可以使用特殊名称来选择特殊的边缘情况格式：

  • best：选择包含视频和音频的单个文件代表的最佳质量格式。
  • worst：选择包含视频和音频的单个文件表示的最差质量格式。
  • bestvideo：选择质量最好的纯视频格式（例如DASH视频）。 可能不可用。
  • worstvideo：选择质量最差的纯视频格式。 可能不可用。
  • bestaudio：选择质量最好的仅音频格式。 可能不可用。
  • worstaudio：选择质量最差的纯音频格式。 可能不可用。

例如，要下载质量最差的纯视频格式，可以使用-f最坏视频。

如果要下载多个视频，而它们没有可用的相同格式，则可以使用斜杠指定首选项的顺序。请注意，斜杠是左关联的，即首选左侧格式，例如-f 22/17/18将下载格式22（如果可用），否则将下载格式17（如果可用），否则将下载格式18（如果有），否则会抱怨没有合适的格式可供下载。 如果您要下载同一视频的多种格式，请使用逗号作为分隔符，例如-f 22,17,18当然会下载所有这三种格式。或更高级的示例结合了优先功能：-f 136/137 / mp4 / bestvideo，140 / m4a / bestaudio。 您也可以通过将条件放在方括号中来过滤视频格式，如-f“ best [height = 720]”（或-f“ [filesize> 10M]”）一样。 以下数字元字段可用于比较<，<=，>，> =，=（等于），！=（不等于）：

  • filesize：字节数，如果事先知道
  • width：视频的宽度（如果已知）
  • height：视频的高度（如果已知）
  • tbr：音频和视频的平均比特率，单位为KBit/s
  • abr：平均音频比特率，单位为KBit/s
  • vbr：平均视频比特率，单位为KBit/s
  • asr：赫兹的音频采样率
  • fps：帧频

还过滤比较的工作=（等于），^ =（以）开头，$ =（以）结尾，* =（包含）以及以下字符串元字段：

  • ext：文件扩展名
  • acodec：正在使用的音频编解码器的名称
  • vcodec：使用中的视频编解码器的名称
  • container：容器格式名称
  • protocol：将用于实际下载的协议，小写（http，https，rtsp，rtmp，rtmpe，mms，f4m，ism，http_dash_segments，m3u8或m3u8_native）
  • format_id：格式的简短说明
  • language：语言代码

任何字符串比较都可以加上否定前缀！为了产生相反的比较，例如！* =（不包含）。 注意，上述元字段都不保证存在，因为这仅取决于特定提取器获得的元数据，即由视频主机提供的元数据。 除非您在运算符后面加上问号（？），否则将排除不知道其值的格式。您可以组合使用格式过滤器，因此-f“ [height <=？720] [tbr> 500]”最多以至少500 KBit / s的比特率选择720p视频（或高度未知的视频）。 您可以使用-f <视频格式> + <音频格式>（需要安装ffmpeg或avconv）将两种格式的视频和音频合并为一个文件，例如，-f bestvideo + bestaudio将仅下载最佳视频文件。格式，最好的纯音频格式，并将其与ffmpeg / avconv混合使用。 格式选择器也可以使用括号进行分组，例如，如果您要下载高度低于480的最佳mp4和webm格式，则可以使用-f'（mp4，webm）[height <480]'。 自2015年4月底和版本2015.04.26起，youtube-dl使用-f bestvideo + bestaudio / best作为默认格式选择（请参阅＃5447，＃5456）(#5447, #5456)。如果安装了ffmpeg或avconv，则将导致分别下载bestvideo和bestaudio并将它们混合到一个文件中，从而提供最佳的整体质量。否则，它会恢复到最佳状态，并导致以单个文件的形式下载最佳质量的文件。非YouTube的视频也需要最好的，因为它们没有以两个不同的文件提供音频和视频。如果只想下载某些DASH格式（例如，如果您不希望获得分辨率高于1080p的视频），则可以在配置文件中添加-f bestvideo [height <=？1080] + bestaudio / best。请注意，如果您使用youtube-dl来流式输出到stdout（然后很可能将其通过管道传输到媒体播放器），即您将输出模板明确指定为-o-，那么youtube-dl仍将使用-f最佳格式选择以立即开始将内容交付给播放器，而不要等到bestvideo和bestaudio被下载并混合后再发送。 如果您要保留旧的格式选择行为（在youtube-dl 2015.04.26之前），即要以单个文件的形式下载质量最好的可用媒体，则应使用-f best明确指定您的选择。您可能需要将其添加到配置文件 configuration file 中，以免每次运行youtube-dl时都将其键入。
